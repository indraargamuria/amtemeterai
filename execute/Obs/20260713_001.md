## Problem Context
- When executing local host debugging via `npm run dev`, `Directory.GetCurrentDirectory()` builds a physical host path (`D:\As a Solution Architect\...`).
- When this string payload is serialized and sent to the Docker adapter API, the Linux container prefixes it, producing an invalid path combination (`/app/D:/As a Solution Architect/...`).
- Additionally, subfolders (`UNSIGNED`, `STAMP`, `SIGNED`) are not being created properly on local runs because path separators and locations are conflicting.
