Task 1 Consolidated Totals in Goods Receipt and SAP Pipeline
Status COMPLETE

Context Modifies the transactional boundaries when a user posts a goods receipt inside DeliveryReceivePage. Currently, the system only streams isolated batch-level and child dataset variables.

Engineering Requirements

Update the state management inside DeliveryReceivePage to calculate aggregate fields across all items: TotalDeliveryQuantity, TotalRejectQuantity, and TotalReturnQuantity.

Adjust the API payload schema sent from the frontend to include these three consolidated high-level fields alongside the existing line item arrays.

Update the backend controller and command handler responsible for receiving this payload.

Modify the database persistence layer to map and save these aggregated totals directly into the corresponding rows within the DeliveryLines table in OpexNOW.

Refactor the SAP outbound integration service layer to map these consolidated totals into the payload structure sent to the SAP gateway during the standard replication sequence. Ensure existing validation pipelines for batch arrays remain unaffected.
