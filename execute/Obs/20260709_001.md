You are an expert Enterprise Systems Engineer and Senior .NET Core Developer. Implement an automated PDF anchor text position extractor using `PdfPig` to dynamically set e-Meterai bounding box stamping coordinates.

## Task 0: Context Inspection
- Analyze `backend-summary.md` to understand the current backend structure
- Acknowledge when done. Do not perform any code modifications yet.

## Task 1: Database Schema Migration
- **Update Invoice Model (`Models/Invoice.cs`):**
  - Add the following nullable integer columns to store the calculated bounding box metrics:
    - `public int? VisLLX { get; set; }`
    - `public int? VisLLY { get; set; }`
    - `public int? VisURX { get; set; }`
    - `public int? VisURY { get; set; }`
    - `public int? StampPageNumber { get; set; }`

## Task 2: Create the PDF Anchor Coordinate Service
- Install the `PdfPig` NuGet package to the backend project.
- Create a new service file at `Services/PdfAnchorService.cs`.
- **Coordinate Calculation Requirements:**
  - Open the PDF stream using `PdfDocument.Open(pdfStream)`.
  - Traverse the PDF pages starting from the last page moving backwards.
  - Search for the keyword string `"Notes"`.
  - When the `"Notes"` text box is located, use the following strict alignment rules:
    1. **Lock Horizontal Target (X-Axis):** Set `visURX = 428` explicitly as a hardcoded parameter.
    2. **Calculate Vertical Target (Y-Axis):** Read the bottom or lower-bound position from the keyword box (`word.BoundingBox.Bottom` or `word.BoundingBox.Top`). Set `visURY = (int)notesY - verticalOffset` to position the stamp slightly below the text line to clear any overlap.
    3. **Derive Remaining Dimensions:** Calculate the lower-left markers relative to your upper-right benchmarks to maintain a uniform 54x54 point bounding box:
       - `visLLX = visURX - 54;`
       - `visLLY = visURY - 54;`
  - Implement a secure fallback to system default constants if the text scanning layer cannot verify the anchor pattern.

## Task 3: Extract Coordinates on Printout Upload
- Open your API controller managing document uploads: `POST /api/invoices/by-number/{invoiceNumber}/upload-printout`
- **Workflow Interlocking:**
  - Once the incoming PDF stream file is processed, pass a readable copy of that stream into your `PdfAnchorService`.
  - Read the returned tuple values (`VisLLX`, `VisLLY`, `VisURX`, `VisURY`, `StampPageNumber`).
  - Update and persist these values directly inside the corresponding target `Invoice` table record row before concluding the database save transaction.

## Task 4: Integrate Saved Coordinates into the Stamping Pipeline
- Open the endpoint handler file for: `POST /api/invoices/{id}/stamp`
- **Refactor Payload Mapping:**
  - Retrieve the target `Invoice` entity tracking record using the incoming path parameters.
  - Read the saved tracking coordinate columns.
  - **Dynamic Interlock:**
    - If the coordinate columns contain values, map them directly into your outgoing Peruri PDS API payload coordinates node (`visLLX`, `visLLY`, `visURX`, `visURY`).
    - If the columns are null, fall back gracefully to your legacy hardcoded template constants.
- Run a clean `dotnet build` to ensure all structural changes compile correctly.
