## Task 2: Legible MinIO Storage Folder Naming Hierarchy

### Context
Currently, document upload endpoints organize file uploads inside MinIO object paths using internal database integer keys (such as `DeliveryID` or `InvoiceID`). To enhance system clarity, these storage paths must use descriptive parameters: `DeliveryNumber` and `InvoiceNumber`.

### Codebase Changes
1. Locate the file attachment endpoints inside your API layer:
   - `POST /api/deliveries/by-number/{deliveryNumber}/upload-printout`
   - `POST /api/invoices/by-number/{invoiceNumber}/upload-printout`
2. Update the logic that generates the prefix destination storage string key passed down to the `IStorageService` client instance.
3. Change the path mapping rules to enforce the following explicit naming conventions:
   - **Deliveries Path:** From `deliveries/{deliveryId}/printouts/filename` to `deliveries/{deliveryNumber}/printouts/filename`
   - **Invoices Path:** From `invoices/{invoiceId}/printouts/filename` to `invoices/{invoiceNumber}/printouts/filename`
4. Confirm that the dynamic key assigned to the `StorageKey` database column maps this exact revised string pattern for subsequent file downloads.

- [x] Task 2 Completed
- [x] Updated backend-summary.md to match the new storage folder paths

## Task 3: Descriptive Prefix File Naming Rules for Uploaded Documents

### Context
When saving printout documents inside their respective storage paths, the actual file string names are currently structured as random raw GUID strings. We need to prefix these filenames with context indicators to simplify audit trails.

### Codebase Changes
1. Return to the upload handlers inside your controller methods for both delivery and invoice upload printouts.
2. Modify the target filename generation string builder logic before the files stream into `IStorageService.UploadAsync`.
3. Configure the naming patterns using these explicit parameters, retaining a unique GUID component at the end to guarantee file collision safety:
   - **Delivery Files:** `DO_{deliveryNumber}_{Guid.NewGuid().ToString()}.pdf`
   - **Invoice Files:** `INV_{invoiceNumber}_{Guid.NewGuid().ToString()}.pdf`
4. Ensure the updated, structured string is evaluated cleanly before the storage engine processes the upload write task.
