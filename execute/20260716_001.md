## Task 4: Deliveries Invoice State Transition Matrix

### Context
The system currently tracks progress via `BillingStatus` in `DeliveryHeaders`[cite: 3]. We need to expose a calculated field called `InvoiceState` on `GET /api/deliveries` and `GET /api/deliveries/{deliveryID}` endpoints while maintaining backward-compatible boolean parameters[cite: 3].

### State Business Logic Engine
1. **Unbilled**: 1st State. Triggered when no invoices exist or the delivery has not initiated billing. Invoice number is returned as `""` or `"-"`. `Invoiced` remains `false`[cite: 3].
2. **Billed**: 2nd State. Triggered when a valid invoice exists that is not voided (`isVoided == false`). Show the active invoice number[cite: 3]. `Invoiced` evaluates to `true`.
3. **Blocked & Voided**: 3rd State. Triggered when the linked invoice has been voided/canceled (`isVoided == true` or `Status == InvoiceStatus.Canceled`)[cite: 3]. Invoice number returns as `""` or `"-"`.
4. **Ready to Re-Billing**: 4th State. Triggered when an invoice has been fully cleared out/voided, transitioning the pipeline back to open optimization eligibility. Invoice number returns as `""` or `"-"`.

### Objectives
1. Implement the state projection calculations within `DeliveriesController.cs` for collection listing and detailed retrieval endpoints[cite: 3].
2. Inject the calculated nodes `invoiceState` (string descriptor matching the four state paths above) and ensure the existing `invoiced` boolean matches the condition (`true` only when state is **Billed**).

### Execution Blueprint
*   Modify `DeliveryResponseDto.cs` and `DeliveryDetailResponseDto.cs` to add:
    ```csharp
    public string InvoiceState { get; set; }
    public string InvoiceNumber { get; set; }
    ```
*   In `DeliveriesController.cs`, query the attached `Invoices` collection to resolve state[cite: 3]:
    ```csharp
    var activeInvoice = delivery.Invoices.FirstOrDefault(i => i.Status != InvoiceStatus.Canceled);
    var canceledInvoices = delivery.Invoices.Where(i => i.Status == InvoiceStatus.Canceled);

    string invoiceState = "Unbilled";
    string invNum = "-";

    if (activeInvoice != null) {
        invoiceState = "Billed";
        invNum = activeInvoice.InvoiceNumber;
    } else if (canceledInvoices.Any()) {
        // Evaluate logic for Blocked & Voided vs Ready to Re-Billing based on internal workflow parameters
        invoiceState = delivery.Status == ReceiverStatus.Canceled ? "Blocked & Voided" : "Ready to Re Billing";
    }
    ```
