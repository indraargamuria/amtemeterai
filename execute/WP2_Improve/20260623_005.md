Target Specification: Environment-Aware Stamping and Path Normalization
This specification introduces environment-driven configurations to PeruriOnPremiseStampService. It ensures cross-platform path building using uniform forward separators and dynamically routes traffic to localhost or container bridges depending on where the app runs.

Task 1: Clean Platform-Agnostic Path Construction
Context
Using string concatenation or generic combining rules on a Windows host creates malformed path strings containing mixed slashes when targeting container storage conventions. We must use explicit forward slashes or safe path combination utilities.

Codebase Changes
Open amtemeterai.Api.Services.PeruriOnPremiseStampService.cs.

Locate your workspace initialization variables.

Replace the Path.Combine structure with a clean, platform-agnostic formatting approach that replaces backslashes with forward slashes universally. This prevents malformed paths on Windows machines while keeping container compatibility:

string stampingSessionId = Guid.NewGuid().ToString("N");
string rawWorkspace = Path.Combine(peruriOptions.SharedFolder, "session" + stampingSessionId);
string containerWorkspace = rawWorkspace.Replace(Path.DirectorySeparatorChar, '/');

string unsignedDirPath = Path.Combine(containerWorkspace, "UNSIGNED").Replace(Path.DirectorySeparatorChar, '/');
string stampDirPath = Path.Combine(containerWorkspace, "STAMP").Replace(Path.DirectorySeparatorChar, '/');
string signedDirPath = Path.Combine(containerWorkspace, "SIGNED").Replace(Path.DirectorySeparatorChar, '/');

Task 2: Implement Automatic Local Development Fallback Routing
Context
When executing locally outside of Docker, the host file system cannot resolve the address signadapter:7777. The application needs an intelligent check to route calls to localhost:9999 if it detects it is running on a local development host.

Codebase Changes
Go to the HTTP client generation section inside amtemeterai.Api.Services.PeruriOnPremiseStampService.cs.

Before making the POST request to the KeyStamp endpoint, insert an environment evaluation check. If the directory root /app/sharefolder does not exist (meaning the code is running directly on your Windows machine), redirect the endpoint request target dynamically to your exposed host port:

string targetUrl = _peruriOptions.KeyStamp;
bool isRunningInDocker = Directory.Exists("/app") || Environment.GetEnvironmentVariable("DOTNET_RUNNING_IN_CONTAINER") == "true";

if (!isRunningInDocker)
{
targetUrl = targetUrl.Replace("signadapter:7777", "localhost:9999");
}

_logger.LogInformation("PHASE 4: Calling KeyStamp Docker adapter at {Url}", targetUrl + "/adapter/pdfsigning/rest/docSigningZ");

Ensure the HttpClient payload configuration sends the request using the evaluated targetUrl string variable.
