Target Specification: Unified Container Topology and Network Resolution
This document specifies a consolidated engineering task to incorporate the Peruri signing adapter service directly into the main docker-compose.yml stack. It aligns internal Docker DNS name resolution with multi-container shared storage mounts to eliminate the network routing exception.

Task 1: Complete Stack Integration and Volume Mapping Synchronization
Context
The backend api container produces an exception when attempting to communicate with http://signadapter:7777 because the service definition layer does not exist inside the current docker-compose runtime topology context. This instruction appends the required container service block and binds them onto a shared data exchange interface.

Detailed Configuration Modifications
Open your docker-compose.yml configuration template.

Navigate down to the api service configuration tree block under the services mapping path. Add the stamping-share volume declaration to its active volumes list array:

volumes:

stamping-share:/app/sharefolder

Move to the bottom of the services structure array, immediately after the createbuckets initialization definition section. Append the complete container orchestration service layout for the signing adapter:

signadapter:
image: peruri-signing-adapter:latest
container_name: signadapter
restart: always
ports:
- "9999:7777"
volumes:
- stamping-share:/app/sharefolder

Note: Change the image property designation string value peruri-signing-adapter:latest to match your local repository container image tag mapping name if it differs.

Proceed to the root-level volumes: layout block at the absolute end of the compose structure. Register the shared volume storage anchor alongside your infrastructure configurations:

volumes:
postgres_data:
minio_data:
stamping-share:

Core Configuration Synchronization Verify
Ensure your options block template class PeruriOptions.cs defaults remain aligned to fallback onto the internal network destinations:
public string KeyStamp { get; set; } = "http://signadapter:7777";
public string SharedFolder { get; set; } = "/app/sharefolder";

Ensure your workspace mapping service worker PeruriOnPremiseStampService.cs cleans backslash path structures cleanly using directory string replacements (.Replace(Path.DirectorySeparatorChar, '/')) to guarantee execution path validity across container borders.

Orchestration Reload Instruction
Apply the topological structure modifications by running a cold restart sequence within your deployment terminal:

docker compose down
docker compose up -d --build
