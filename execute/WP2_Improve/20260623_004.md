Target Specification: Docker Container Network Alignment and Shared Volume Stamping

This document outlines two sequential technical updates required to transition the stamping system from local host bindings to native docker container-to-container communication, solving the external path visibility issue completely using a shared Docker Named Volume. Follow the tasks sequentially.

Task 1: Orchestration Infrastructure and Shared Named Volume Configuration

Context:
To allow the api container and the signadapter container to seamlessly exchange unsigned documents, QR codes, and signed PDFs, both containers must be configured to use a single, shared Docker Named Volume instead of conflicting temporary paths. The API will also communicate directly with the adapter container using internal Docker DNS.

Docker Compose Changes:

Open your docker-compose.yml file.

Under the root level volumes block at the very bottom of the file, declare a new shared volume called stamping-share alongside postgres_data and minio_data:

volumes:
postgres_data:
minio_data:
stamping-share:

Locate the api service block inside your services declaration layout.

Update the Peruri__KeyStamp environment variable value inside the api service block to point directly to the internal container endpoint instead of localhost:

Peruri__KeyStamp: http://signadapter:7777

Add the volume mount reference inside the volumes list array under the api service container block:

volumes:

stamping-share:/app/sharefolder

Ensure your signadapter container service block is added to the services tree (or adjusted if it exists) using the explicit container name signadapter and matching internal port. Map the same shared volume mount to it:

signadapter:
image: your-signing-adapter-image:latest
container_name: signadapter
restart: always
ports:
- "9999:7777"
volumes:
- stamping-share:/app/sharefolder

Task 2: Codebase Option Fallbacks and Stamping Workspace Path Alignment

Context:
The C# options fallback endpoints and directory targets must be aligned to point to the shared volume space mounted at /app/sharefolder inside the container ecosystem rather than the isolated host system temporary folder path.

Codebase Changes:

Open your PeruriOptions.cs class file located inside your configuration namespace layout (amtemeterai.Api.Config).

Change the default property value initialization fallback for KeyStamp from http://localhost:9999 to use the internal Docker container host address:

public string KeyStamp { get; set; } = "http://signadapter:7777";

Change the default property value initialization fallback for SharedFolder from /sharefolder to point explicitly to the mounted container path directory:

public string SharedFolder { get; set; } = "/app/sharefolder";

Open your stamping worker execution handler file at amtemeterai.Api.Services.PeruriOnPremiseStampService.cs.

Locate the execution phase blocks where target storage directories are calculated and initialized. Remove any logic binding to Path.GetTempPath().

Replace it with the configured options property value (which maps to /app/sharefolder), combined with a unique tracking subdirectory for the active transaction payload to avoid data race condition overrides:

string stampingSessionId = Guid.NewGuid().ToString("N");
string containerWorkspace = Path.Combine(peruriOptions.SharedFolder, "session" + stampingSessionId);
string unsignedDirPath = Path.Combine(containerWorkspace, "UNSIGNED");
string stampDirPath = Path.Combine(containerWorkspace, "STAMP");
string signedDirPath = Path.Combine(containerWorkspace, "SIGNED");

Ensure that the dynamic tracking directory tree is created before document streaming initialization:

Directory.CreateDirectory(unsignedDirPath);
Directory.CreateDirectory(stampDirPath);
Directory.CreateDirectory(signedDirPath);

Confirm that the cleanup loop inside your finally block remains functional to wipe the session directory from the shared volume storage context smoothly upon successful MinIO streaming:

finally
{
if (Directory.Exists(containerWorkspace))
{
Directory.Delete(containerWorkspace, true);
}
}
